User Guide

系统使用说明

从参考素材到平台表格，按实际业务步骤完成任务创建和导出。

国内版本怎么用

国内版本按聚水潭 22 列商品导入模板组织数据。系统按商品事实生成每套商品元数据和图片，再按 SKU 组合写入款式编码、商品编码、颜色、规格、库存与价格。

  1. 进入 创建(聚水潭)，填写内部任务名称和商品事实；有现成标题方向时再填写可选参考标题。
  2. 上传核心参考图和辅助参考图；核心参考图决定商品主体，辅助参考图用于补充角度、材质、细节。
  3. 选择无规格、一维规格或二维规格。颜色、款式等需要图片的维度放一级规格，尺码、容量等放二级规格。
  4. 逐个 SKU 组合填写库存、基本售价、市场价、商家 SKU 和可选分销控价；价格浮动也按组合独立设置。
  5. 选择 SKU 图片自上传或 AI 生成模式，再点击 创建并生成。自上传模式不会创建 SKU 图片工作流。
  6. 任务完成后导出，系统会保留模板说明和表头，从第 3 行写入数据。
模板依据：系统内置 /templates/聚水潭_商品模板.xlsx 的“商品导入模板”工作表。第 1 行是导入说明，第 2 行是 22 列表头，第 3 行开始写商品数据。

国内字段填写建议

字段怎么填影响
任务名称给自己识别批次用，例如“手机壳-透明款-7月”。只用于任务列表和文件识别，不发送给工作流。
商品事实必填品类、材质、功能、尺寸、包装、适用场景和不能虚构的信息。10~2000 字符，是每套标题、推荐文案、商品描述和 SKU 标题的事实边界。
参考标题可选，提供已有标题、核心关键词或标题方向。只作生成参考；与商品事实冲突时以商品事实为准。
参考图核心图必须上传；辅助图可上传不同角度、包装、细节。只作为 AI 输入，不直接写入最终导出表。
商品结构没有颜色尺码选无规格；只有颜色选一维；颜色加尺码选二维。决定模板“颜色”和“规格”两列以及 SKU 组合数量。
库存与价格每个 SKU 组合独立填写。市场价和分销控价可留空。分别写入库存、基本售价、市场|吊牌价和分销控价列。
商家 SKU可填写短编码，例如 RED-M；留空时系统自动使用 V1、V2。同一任务内不能重复。系统会加上每套款式编码前缀，确保商品编码唯一。
生成套数需要多少个商品款式就填多少套。每套会生成独立标题和商品图片。
价格浮动默认关闭；开启后给单个变体填写最低价、最高价和取价精度。每套商品导出时会在区间内稳定取价；精度可选 2/1/0 位小数，或十位/百位归零。
全量生成每套都生成完整主图、详情图、SKU 图。成本更高，但差异最大。
去重生成多套商品共享部分通用图，只保留必要差异。适合只换封面或轻微差异的批量生成。
SKU 图片模式默认自上传；需要模型出图时切换 AI 生成。每个一级规格必须上传一张。自上传图直接导出；AI 模式将上传图作为参考并导出生成图。同一颜色下的 S/M/L 共用 URL。
商品链接/供应商均为模板可选字段，按业务需要填写。原样写入每个 SKU 行，不进入 AI 生成。

国内图片理解

  • 模板“商品主图”包含系统生成的主图和附图，并以 JSON 数组写入。
  • “商品详情图”是 3:4 的详情页图片序列，不写入独立的“3:4主图”列。
  • 商品参考图不是最终资源；SKU 自上传模式是例外，上传后的 R2 URL 会直接写入图片地址列。
  • 导出只检查实际已构造的工作流资源；管理员关闭商品元数据或 SKU 图片工作流时，不会等待未构造的计划。

Shopee 版本怎么用

Shopee 版本按官方 mass upload template 组装 Excel。你只提供可以确认的商品事实、参考图、规格和交易信息，系统生成越南语商品标题、商品描述与商品套图，再把 R2 URL 和规格数据写入模板。

  1. 上传 1 张核心参考图；需要补充角度、结构或包装时再添加辅助图。
  2. 填写商品事实，包括品类、材质、功能、适用场景、包装内容和不能虚构的信息。系统据此生成每套越南语标题与描述。
  3. 选择无规格、一维规格或二维规格。需要图片区分的颜色、图案、款式放在一级规格，尺码等无图维度放在二级规格。
  4. 填写每个 SKU 组合的价格、库存、重量和可选商家 SKU。重量默认 0.2kg，可按 SKU 修改。
  5. 选择 SKU 图片自上传或 AI 生成模式；选择至少一个物流渠道。SPX 默认开启，库存默认 999。
  6. 点击 创建并生成。系统固定生成每套 1 张封面和 8 张附图，不构造 Shopee 详情图计划。
  7. 任务完成后导出官方模板，数据从第 7 行开始写入。需要归档全部图片时可在任务列表打包下载。
官方模板依据：系统内置 /templates/Shopee_mass_upload_2026-07-14_basic_template.xlsx 的 Template 工作表。第 3 行是字段名，第 4 行是必填性，第 6 行是填写规则，第 7 行开始写商品数据。

Shopee 创建页字段说明

字段填写建议官方/系统规则
商品事实必填。写品类、材质、功能、规格、适用场景、包装内容和限制，避免只写关键词。10~2000 字符，只作为 AI 事实边界。每套 Product Description 由元数据工作流生成，官方要求 100~3000 字符。
任务名称必填内部批次备注，例如“手机壳-越南站-透明款”。只用于任务列表和文件识别，不发送给工作流，也不作为最终 Product Name。
参考标题可选，提供标题方向或已有关键词。用于生成 Product Name，不直接导出；事实冲突时以商品事实为准。
主图生成描述描述封面和附图的视觉方向。影响 main_1 和 sub_2~sub_N。Shopee 默认生成 9 张，即 1 张封面图 + 8 张附图。
生成模式每套都要独立图片选全量；同款只换封面选去重。去重模式下每套生成 1 张独立封面，第 2 套开始复用第 1 套的 8 张附图和 SKU 图。
生成套数要生成多少套商品资源就填多少。导出行数 = 生成套数 × 变体数。
分类 ID建议从 Shopee Seller Center 的 Category Tree 查。模板说明：可从 Category Tree 选择；留空时 Shopee 可能推荐类目。准确类目有利于搜索和属性匹配。
重量(kg)在每个 SKU 组合中填写，默认 0.2kg。官方模板 Z Weight 单位是 g，系统逐 SKU 从 kg 转成 g，必须大于 0 且不超过 100000000g。
长/宽/高(cm)按包裹尺寸填。用于物流费计算；官方提示尺寸不准会影响运费和履约。
一级规格名优先放需要图片区分的维度，通常是颜色、图案或款式。官方要求 1~14 字符；Image per Variation 只映射一级规格值。
规格值1例如 Red、Blue、Black。官方要求 1~20 字符，SKU 变体图映射到规格值1。
二级规格名/值放不需要独立图片的维度，例如 Size / S、M、L。启用后系统自动生成笛卡尔组合，可逐个关闭不售卖组合；二级规格不会新增图片工作流。
规格图模式默认“自上传”；需要模型生成新图时选择“AI 生成”。upload 直接导出用户 R2 URL且不创建工作流；ai 每套按唯一一级规格创建 SKU 图计划。
价格按越南站 VND 填数字；也可在单个变体启用价格浮动。当前模板规则为 1000~120000000；浮动价格会按每套商品在区间内取值。
库存默认 999，可按每个 SKU 组合修改。官方范围 0~10000000；用户填写 0 时系统会保留 0。
SKU店铺内唯一，建议简短；同一任务内不能重复。官方要求小于 100 字符；系统会加上套图父 SKU 前缀防止跨套重复。
SKU 图片每个一级规格必须上传一张，不要按尺码重复上传。自上传模式上传成品图，AI 模式上传参考图。自上传模式直接导出；AI 模式最终导出生成后的 sku_N。同一一级规格的组合行复用 URL。
物流渠道默认勾选 SPX，可按店铺实际渠道调整。新版模板要求每个商品至少一个渠道为 On；已移除 5115，并新增 50053（最高价 3000000 VND）。

分类 ID 和品牌

分类 ID 是什么

分类 ID 是 Shopee 平台的类目编码，不是系统自定义字段。官方模板提示应从 Seller Center 的 Category Tree 选择分类 ID；准确分类有利于搜索结果和后续类目属性匹配。

官方样例中出现过 120039、120055 这类数字，但样例不是通用类目映射表，不要直接复制。实际类目应以当前站点 Seller Center 下载的 Category Tree 或类目模板为准。

如果暂时不知道分类 ID，可以留空。官方模板写明系统会推荐类目；但留空会降低可控性，类目属性、物流、价格限制可能需要上传后再处理。

品牌和“无品牌”

当前系统导出使用 Shopee 2026-07-14 basic template 的 A~AL 列，这个模板没有 Brand 列，因此创建页不要求品牌 ID，也不会在导出的 Excel 中虚构品牌。

Shopee 的类目专用模板或样例表里 Brand 通常要求填写 valid brand 或 no brand；样例规则还写明：可查品牌表的 valid brand id，或者留空后在 attributes tool 里补属性值。

本仓库这份官方模板的隐藏品牌表没有可读品牌清单，也没有给出统一的“无品牌 ID”。因此系统不会默认填一个无品牌 ID。后续切换类目专用模板时，应从 Seller Center 或官方模板品牌表选择有效品牌或 no brand。

导出前检查

  • 任务状态已完成，推送计划没有 pending/processing。
  • 每套都有 AI 商品标题和商品描述，长度分别符合 10~120、100~3000。
  • 每套都有 main_1 封面和 sub_2~sub_N 附图。
  • 规格名和值已经规范化；启用规格图时，每个一级规格还有生成后的 SKU 图。
  • 至少选择一个物流渠道。
  • 价格、库存、重量、规格长度符合官方模板限制。
