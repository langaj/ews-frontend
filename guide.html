系统使用说明

国内版本怎么用

国内版本面向聚水潭上传逻辑。系统先按任务生成商品套图，再把每套图、变体、标题、SKU 标题组装成导出表。

  1. 进入 创建(聚水潭)，填写任务名称、商品描述、生成套数、生成模式。
  2. 上传核心参考图和辅助参考图；核心参考图决定商品主体，辅助参考图用于补充角度、材质、细节。
  3. 填写颜色、尺码、价格、库存、SKU 等变体信息。
  4. 提交任务后在任务列表点击推送，等待标题、SKU 标题、主图、详情图、SKU 图全部完成。
  5. 完成后导出聚水潭表格，导出的图片 URL 来自系统保存到 R2 图床的生成结果。

国内字段填写建议

字段怎么填影响
任务名称给自己识别批次用，例如“手机壳-透明款-7月”。影响任务列表展示，也会作为 AI 生成标题的参考。
参考图核心图必须上传；辅助图可上传不同角度、包装、细节。只作为 AI 输入，不直接写入最终导出表。
生成套数需要多少个商品款式就填多少套。每套会生成独立标题和商品图片。
全量生成每套都生成完整主图、详情图、SKU 图。成本更高，但差异最大。
去重生成多套商品共享部分通用图，只保留必要差异。适合只换封面或轻微差异的批量生成。
颜色/尺码颜色是一级规格，尺码是二级规格；没有尺码可留空。决定 SKU 组合。
SKU 参考图每个颜色上传一张参考图。用于生成变体图和 SKU 标题。

国内图片理解

  • 国内导出里“主图”通常包含系统生成的主图/附图，按平台上传工具要求整体使用。
  • “详情图”是商品详情页长图或详情模块图片。
  • 参考图不是最终资源；最终导出的图片必须是 AI 生成并保存到 R2 后的 URL。

Shopee 版本怎么用

Shopee 版本按官方 mass upload template 组装 Excel。系统不是直接把你上传的图片写进表格，而是先生成商品套图，再把生成后的 R2 URL 写入模板。

  1. 进入 创建(虾皮)，填写任务名称、商品描述、参考标题。
  2. 上传核心参考图和辅助参考图；这些图只作为 AI 生成输入。
  3. 填写生成套数、生成模式、分类 ID、重量、尺寸、规格名和变体。
  4. 每个变体上传 SKU 参考图，系统用它生成 Shopee 的变体图片。
  5. 选择至少一个物流渠道。
  6. 提交后推送任务，默认等待商品标题、SKU 标题、封面图、附图、SKU 图完成；只有把详情图数量设为大于 0 时才会额外生成详情图。
  7. 完成后导出 Shopee 官方模板，数据从第 7 行开始写入。
官方模板依据：系统内置 /templates/Shopee_mass_upload_2026-07-03_basic_template.xlsx 的 Template 工作表。第 3 行是字段名，第 4 行是必填性，第 6 行是填写规则。

Shopee 创建页字段说明

字段填写建议官方/系统规则
任务名称内部批次名，例如“手机壳-越南站-透明款”。用于任务列表，也会传给 AI 作为标题参考。
商品描述写材质、功能、包装、适用场景，尽量完整。Shopee 模板要求 Product Description 100~3000 字符。
参考标题放你希望 AI 学习的标题风格，可以多行。用于生成 Product Name 和 SKU 标题，不直接导出。
主图生成描述描述封面和附图的视觉方向。影响 main_1 和 sub_2~sub_N。
详情图生成描述写卖点、结构、场景、参数表达。仅在详情图数量大于 0 时生效；当前 basic template 暂不导出详情图，可在任务列表打包下载。
详情图数量默认 0。只有需要额外归档详情素材时才填 1~9。Shopee basic template 没有详情图字段，填 0 时系统不会要求详情图工作流。
生成模式新款差异大选全量；同款换封面/颜色选去重。去重模式下第 2 套开始复用第 1 套附图、SKU 图；详情图数量大于 0 时也复用详情图。
生成套数要生成多少套商品资源就填多少。导出行数 = 生成套数 × 变体数。
分类 ID建议从 Shopee Seller Center 的 Category Tree 查。模板说明：可从 Category Tree 选择；留空时 Shopee 可能推荐类目。准确类目有利于搜索和属性匹配。
品牌 ID当前可留空。当前 basic Template A~AP 没有 Brand 列，系统不导出品牌 ID。类目专用模板中 Brand 可能必填，应使用有效品牌或 no brand。
重量(kg)按实际商品包裹重量填写 kg。官方模板 AD Weight 单位是 g，系统导出时自动从 kg 转成 g，范围 0~100000000g。
长/宽/高(cm)按包裹尺寸填。用于物流费计算；官方提示尺寸不准会影响运费和履约。
规格名1通常填颜色、款式。官方要求 1~14 字符。
规格值1例如 Red、Blue、Black。官方要求 1~20 字符，SKU 变体图映射到规格值1。
规格名2/规格值2有尺码时填写，例如 Size / M。存在二级规格时必须成对填写。
价格按越南站 VND 填数字。当前模板规则为 1000~120000000，最高价/最低价比例不超过 5。
库存每个变体库存。官方范围 0~10000000。
SKU店铺内唯一，建议简短。官方要求小于 100 字符；系统会加上套图父 SKU 前缀防止重复。
SKU 参考图每个一级规格上传一张清晰参考图。只作为 AI 输入，最终导出使用生成后的 sku_N。
物流渠道至少勾选一个可用渠道。官方模板要求每个商品至少一个渠道为 On；系统导出未选渠道为 Off。

分类 ID 和品牌 ID

分类 ID 是什么

分类 ID 是 Shopee 平台的类目编码，不是系统自定义字段。官方模板提示应从 Seller Center 的 Category Tree 选择分类 ID；准确分类有利于搜索结果和后续类目属性匹配。

官方样例中出现过 120039、120055 这类数字，但样例不是通用类目映射表，不要直接复制。实际类目应以当前站点 Seller Center 下载的 Category Tree 或类目模板为准。

如果暂时不知道分类 ID，可以留空。官方模板写明系统会推荐类目；但留空会降低可控性，类目属性、物流、价格限制可能需要上传后再处理。

品牌 ID 和“无品牌”

当前系统导出使用 Shopee basic template 的 A~AP 列，这个模板没有 Brand 列，所以创建页的品牌 ID 目前是预留字段，不会写入导出的 Excel。

Shopee 的类目专用模板或样例表里 Brand 通常要求填写 valid brand 或 no brand；样例规则还写明：可查品牌表的 valid brand id，或者留空后在 attributes tool 里补属性值。

本仓库这份官方模板的隐藏品牌表没有可读品牌清单，也没有给出统一的“无品牌 ID”。因此系统不会默认填一个无品牌 ID。当前建议：品牌 ID 留空；如果后续使用类目专用模板，再从 Seller Center/官方模板品牌表选择有效品牌或 no brand。

导出前检查

  • 任务状态已完成，推送计划没有 pending/processing。
  • 每套都有 AI 商品标题，且长度符合 10~120。
  • 每套都有 main_1 封面和 sub_2~sub_N 附图。
  • 每个变体都有 AI SKU 标题和生成后的 SKU 图。
  • 至少选择一个物流渠道。
  • 价格、库存、重量、规格长度符合官方模板限制。
